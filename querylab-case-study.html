QueryLab

A PostgreSQL Behavior Laboratory

❓ The Problem: The "Black Box" Database

Most developers write SQL but don't know what happens under the hood. QueryLab solves the mystery of:

  • Execution Path: Seeing exactly why the database chooses a Sequential Scan over an Index Scan.
  • Index Impact: Observing the latency trade-offs when adding multiple indexes.
  • Selectivity: Visualizing how data distribution changes the database's strategy.

🛠️ The Build: An Interactive Lab

Plan Parser

Node.js logic to simplify EXPLAIN ANALYZE JSON output.

Lab Modes

Seeding and skewing 200k+ rows for real experiments.

Caching Layer

Integrated Redis for comparative performance benchmarking.

The heart of QueryLab is a custom parser that takes raw EXPLAIN output and highlights "Workhorse" nodes, showing users the difference between efficient and inefficient scans.

🧠 Technical Deep Dive: Composite Indexing Lab

One of the most powerful simulations focuses on the Left-to-Right Rule. Users seed a table with 1M rows and test queries against different index types.

The lab visually proves why the order of columns in a composite index matters for performance by showing the exact execution costs for each scenario.

✨ The Results

  • Technical Transparency: No more guessing; observe exactly how PostgreSQL sees your data.
  • Experimental Freedom: A safe environment to "break" things and see the performance consequences.
