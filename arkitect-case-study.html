Arkitect

Real-Time System Design Simulator

❓ The Problem: Theoretical Exhaustion

System design is often taught using static diagrams, making it hard to grasp the dynamic nature of distributed systems:

  • Cascading Failures: How one "down" database replica can bring down an entire cluster.
  • LB Algorithms: The actual latency difference between Round Robin and Least Connections under load.
  • Traffic Spikes: What happens to queue depth when RPS jumps 10x instantly.

🛠️ The Build: A Real-Time Engine

Simulation Core

Discrete event simulator written in high-performance Go.

Interactive UI

React dashboard with React Flow for node visualization.

Streaming Metrics

Live telemetry via WebSockets (gorilla/websocket).

I built the heart of Arkitect as a custom discrete event simulator in Go. It calculates p95 latency, error rates, and throughput on every simulated request "tick," streaming results to the frontend every 500ms.

🧠 Technical Deep Dive: Congestion & Failure

The most complex part was modeling realistic congestion using Queue Theory. Servers have finite concurrency; once full, requests queue up, increasing p99 latency exactly like a real production environment.

I also implemented active failure injection, allowing users to toggle "Node Down" states to observe how a Load Balancer detects and routes around a dead server.

✨ The Results

  • Real-time Visualization: Instant feedback on architectural changes.
  • Infinite Scenarios: Simulate everything from simple monoliths to complex sharded clusters.
